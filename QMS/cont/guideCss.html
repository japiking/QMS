3. CSS

3.1. CSS 작성규칙

  • CSS는 2.0을 기준으로 인코딩은 HTML과 동일하게 작업한다.
  • CSS의 가독성을 위해 의미없는 줄바꿈을 하지 않는다.
  • 빈 줄은 두 줄 이상 띄우지 않는다.
  • 0을 제외한 모든 수치에는 단위(px, %)를 붙인다.
  • 색상값은 RGB값 6자리(예:#000000)로 지정한다. 키워드(예:black) 값은 사용하지 않는다.
  • 주석기호 사이 앞뒤로 한 칸씩 공백 이외 불필요한 공백을 넣지 않는다. 예) /* 주석 */
  • 속성값은 큰 따옴표(“”)로 묶는다.
  • CSS 속성 선언순서는 다음과 같은 순서로 사용한다. {display/ overflow/ float/ position/ width/ height/ padding/ margin/ border/ 글꼴; 배경; …}
    position과 연관된 속성의 순서는 top, right, bottom, left(시계방향), z-index 순으로 한다.
  • 한글 폰트 속성값은 “돋움”, dotum과 같이 한글과 영문을 모두 표기한다.

3.2. CSS 파일구조

  • bBase.css : CSS의 Base setting
  • bLayout.css : 페이지의 레이아웃 관련 css (기업온라인에서는 header, footer 영역의 내용을 bHeadFoot.css 에 따로 분리해 둠)
  • bHeadFoot.css : 개인뱅킹,기업뱅킹, 외환포탈 등에서 공통으로 사용하는 header, footer 스타일의 내용을 가지는 css
  • bCommon.css : 각종 템플릿 및 공통으로 사용가능한 css
  • bTable.css : 페이지의 모든 table 관련 css
  • bContents.css : bBase.css / bCommon.css / bLayout.css /bHeadFoot.css/ bTable.css 를 import하는 sub 관련 css
  • bMain.css : bBase.css / bCommon.css / bLayout.css / bHeadFoot.css 를 import하고 있는 main 관련 css
  • bPopup.css : bBase.css / bCommon.css / bTable.css 를 import하고 있는 popup 관련 css

3.3. CSS 사용

3.3.1. bContents.css - 서브화면

@import url('bBase.css'); @import url('bCommon.css'); @import url('bLayout.css'); @import url('bHeadFoot.css'); @import url('bTable.css'); 서브 컨텐츠 화면 스타일

3.3.2. bMain.css - 메인화면

@import url('bBase.css'); @import url('bCommon.css'); @import url('bLayout.css'); @import url('bHeadFoot.css'); 메인 화면 스타일

3.3.3. bPopup.css - 팝업화면

@import url('bBase.css'); @import url('bCommon.css'); @import url('bTable.css'); 팝업 컨텐츠 스타일
